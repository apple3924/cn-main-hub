# cn-main-hub

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库本身不针对任何单一域名或具体网站，仅作为页面资源的集中存放与分发点。

## 项目简介

本仓库旨在收集、整理并托管若干独立的 HTML 页面。这些页面可能来自不同时期、不同用途，但统一归档于此，便于维护、版本管理以及通过 GitHub Pages 进行访问。

## 目录说明

```
.
├── index.html          # 仓库首页/导航页（可选）
├── pages/              # 存放各独立 HTML 页面
│   ├── page-a.html
│   └── page-b.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本说明文件
```

- `pages/`：所有独立 HTML 页面均放置于此目录下，每个页面文件名应具有辨识度。
- `assets/`：页面所依赖的样式、脚本、图片等公共资源。若页面自带内联资源，也可不依赖此目录。
- `index.html`：可选的导航页，用于列出所有归档页面并提供链接。

## 页面归档说明

- 每个页面均为独立完整的 HTML 文件，可脱离仓库环境直接运行。
- 页面之间无强依赖关系，但可共享 `assets/` 中的公共资源。
- 归档页面可能包含历史版本，建议在文件名或目录内标注版本信息（如 `page-v2.html`）。
- 本仓库不保证页面内容的时效性或准确性，仅作存档用途。

## 维护说明

- 欢迎提交 Pull Request 添加新的独立页面或更新现有页面，但请确保页面内容不违反平台政策。
- 添加页面时，请一并更新 `index.html`（若存在）中的链接列表。
- 不建议在本仓库中存放动态后端或需要服务器解释执行的文件（如 PHP、Python 脚本等）。
- 若页面涉及外部资源（如 CDN 库），请尽量使用可靠来源，并注意版本兼容性。

## 使用方式

1. 克隆本仓库到本地。
2. 直接打开 `pages/` 下的任意 HTML 文件即可查看。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能直接访问归档页面。
